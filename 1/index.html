Project 1 — Colorizing the Prokudin-Gorskii Collection

Overview

The goal of this project is to take the digitized Prokudin-Gorskii glass plate images and automatically produce a color image. The input is a single grayscale plate stacked vertically in the order B, G, R. I split the image into three equal parts, then align the G and R channels to B.

Approach

To align channels, I implemented a pyramid search: start at low resolution for efficiency, then refine offsets at higher resolutions. This allows large displacements to be found quickly while keeping runtime reasonable.

For similarity scoring, I tested multiple options. NCC worked fine, but I also implemented Canny edge features for robustness in cases where brightness differs across channels.

After alignment, I apply simple post-processing steps:

  • Auto-contrast — stretch intensities so the darkest pixel is 0 and the brightest is 1.
  • White balance — normalize means of the three channels (gray-world assumption).
  • Automatic cropping — detect and remove noisy borders based on variance and pixel sums.

Problems & Failures

Some images did not align perfectly. For example, Emir has very different brightness levels between channels, making intensity-based metrics fail. Using edge-based alignment improved results, but some misalignment still remains.

Borders are also difficult to handle because they contain strong edges that do not correspond between channels. Automatic cropping helps reduce this effect.

Bells & Whistles

Edge-based Alignment

I use Canny edge detection to align difficult cases such as Emir, where brightness differences cause failures in raw pixel metrics.

Automatic Cropping

By measuring row and column variance, I detect low-information borders and trim them away automatically.

White Balance

I normalize channel means under the gray-world assumption to reduce color cast and improve overall balance.
