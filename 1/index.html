Project 1 — Colorizing the Prokudin-Gorskii Collection

Overview

The goal of this project is to take the digitized Prokudin-Gorskii glass plate images and automatically produce a color image. The input is a single grayscale plate stacked vertically in the order B, G, R. I split the image into three equal parts, then align the G and R channels to B.

Approach

To align channels, I implemented a pyramid search: start at low resolution for efficiency, then refine offsets at higher resolutions. This allows large displacements to be found quickly while keeping runtime reasonable.

For similarity scoring, I tested multiple options. NCC worked fine, but I also implemented Canny edge features for a more robust alignment.

Besides simple alignment, I also implemented these bells and whistles:

  • Automatic cropping — Crops the image borders by filtering out low variance or low mean value rows/cols; done after alignment.
  • Automatic contrasting — Maps the darkest pixel to 0 and the brighest to 1 and applied linear transformation on all pixels; done before alignment.
  • Automatic white balance — Assumes gray-world illumination to shift all 3 colors towards the gray mean; done before alignment.
  • Better color mapping — Normalizes the average pixel value to gray; done before alignment.
  • Better features — Uses canny filter for alignment instead of NCC; used during alignment.

Problems & Failures

The biggest problem I encountered is border detection. I tried multiple methods: detect boarder based on variance, intensity, or gradient, and every combination between these three. However, no matter how hard I try, I cannot detect border perfectly for every image because of the noise on the border (some has numbers on the border, some has stains, etc.). The final approach is to use a mix of all three methods, find the border on each r,g and b image, then find the IoU region as the final result. Although borders are still visible on some images, borders are much smaller than without applying the algorithm.
